Dev notes:
3-layer separation 
- taskStore only knows about reading/writing raw JSON. 
- taskController owns all validation and HTTP responses. 
- routes/tasks.js is pure Express wiring, nothing else.
Immutable fields 
- PATCH silently strips id and date even if sent in the body, so those can never be overwritten.
 Storage path 
- node-localstorage writes to ./data/localStorage/ on disk. 
- No cookies, no sessions, no in-memory-only state — data survives server restarts.
date is set once at creation 
- it's the task's creation timestamp, not a mutable field.
